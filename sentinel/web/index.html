    SYSTEM WALKTHROUGH
    

            
    
                
    
                    
    1. EXECUTE & PROVE
    
                    
    When you run a model via the CLI, SentinelRun wraps the execution. It hashes the Model and Input,
                        captures the Output, and signs it all into a Cryptographic Proof.
    
                    
    
                        $ python -m sentinel.cli run model.bin "Input" --store
                    
    
                
    

                
    
                    
    2. IMMUTABLE STORAGE
    
                    
    The Proof is uploaded to IPFS (InterPlanetary File System). It gets a unique
                        Content Identifier (CID). This CID is the permanent address of that specific
                        execution trace.
    
                
    

                
    
                    
    3. VERIFY ANYWHERE
    
                    
    Anyone with the CID can independently verify the execution. The Verifier checks:
    
                    
      
                        
    • Does the hash of the trace match the signed proof?
      • 
                        
    • Was the model allowed by policy?
      • 
                        
    • Has the output been tampered with?
      • 
                    
    
                
    

                
    
                    
    4. TRY IT NOW
    
                    
    Make sure the backend is running. Run a CLI command to get a CID. Paste the CID into the search
                        bar on this dashboard to visualize the proof.
